# 后续更新规则

以后继续这个项目时，所有新增内容都必须进入本项目文件夹对应目录。

## 新增一个 agent 时

按下面位置更新：

| 内容 | 放置位置 |
| --- | --- |
| agent 实现 | `src/water_ai/agents/agent_name.py` |
| 单测 | `tests/test_agent_name.py` |
| 运行脚本 | `experiments/run_agentXX_agent_name.py` |
| 运行报告 | `outputs/agentXX_agent_name/` |
| 成果索引 | `deliverables/manifest.json`、`deliverables/artifact_index.md` |
| 状态说明 | `notes/current_status.md` |
| 迭代记录 | `notes/iteration_log.md` |

## 更新研究方案或汇报材料时

| 内容 | 放置位置 |
| --- | --- |
| Word/项目书正文 | `docs/` |
| PPTX | `deliverables/ppt/` |
| PPT 设计与 QA | `deliverables/deck_*.md` |
| 汇报提纲和讲稿 | `deliverables/presentation_outline.md`、`deliverables/slide_narrative_script.md` |
| 关键指标口径 | `deliverables/key_metrics_table.md` |

## 更新模型时

| 内容 | 放置位置 |
| --- | --- |
| 模型训练脚本 | `experiments/train_soft_sensor_model.py` 或新训练脚本 |
| 模型文件 | `models/` |
| 训练数据与指标 | `outputs/soft_sensor_training/` |

## 更新真实数据校准时

| 内容 | 放置位置 |
| --- | --- |
| 现场数据接口规范 | `docs/field_data_interface_spec.md` |
| CSV 模板和数据包 | `outputs/agent30_field_data_interface/` |
| 现场校准协议 | `deliverables/field_calibration_protocol.md` |
| 数据验收门 | `deliverables/field_data_acceptance_gates.md` |
| 校准运行手册 | `deliverables/field_calibration_runbook.md` |

## 禁止事项

- 不再在 `py学习` 根目录创建本项目的 `outputs`、`low_cost_water_plan_*`、单独 docx 或临时渲染文件夹。
- 不把 synthetic/sample 数据写成现场实证结论。
- 不在 PPT 已冻结时反复改展示层，除非明确要求。
