## 目录框架

| 目录 | 作用 | 后续更新规则 |
| --- | --- | --- |
| `00_项目入口与说明` | 项目导航、阅读顺序、后续更新规则 | 只放说明和入口，不放运行产物 |
| `docs` | Word 方案、系统规格、研究方案文档 | 方案、规范、项目书正文更新到这里 |
| `deliverables` | 汇报交付材料、成果索引、PPT 相关材料 | 对外展示、PPT、整理成果更新到这里 |
| `src` | Python 包和所有 agent 代码 | 新 agent 实现放 `src/water_ai/agents/` |
| `experiments` | 单 agent 和全链条运行脚本 | 新 agent 脚本按 `run_agentXX_*.py` 命名 |
| `tests` | 回归测试 | 新 agent 必须配对应测试 |
| `models` | 本地模型文件 | 训练出的模型放这里 |
| `outputs` | 运行报告、模拟结果、agent 报告 | 新 agent 报告放 `outputs/agentXX_*` |
| `notes` | 当前状态和迭代日志 | 每次阶段更新后同步这里 |

## 重要边界

- `outputs/agent30_field_data_interface/synthetic_field_data_package/` 只能用于接口演示和脚本联调。
- 真实校准必须先通过 Agent34 的 `G0_data_origin` 门。
- 真实 timestamped replay 必须先通过 Agent44 的 metadata/provenance、CSV 字段、类型转换和 batch 关联门，再按 Agent45 证据链进入 Agent42/43；不能单独绕过入口门。
- 不复制 `.venv`，避免把环境缓存当项目成果；需要运行时按 `requirements.txt` 重建环境。
