# 视觉故事板

## S1 低成本传感条件下，把水处理黑箱变成灰箱

- 页面目的：建立研究问题和核心命题。
- 视觉素材：`grey_box_loop` 黑箱到灰箱逻辑图
- 讲述重点：贵仪器和慢检测不是简单替换问题，而是需要系统结构为推断争取时间。

## S2 总体方案：循环结构 + 软传感器 + 多智能体闭环

- 页面目的：说明系统为什么不是一次处理，而是动态回流、暂存、加药、验证和放行。
- 视觉素材：`control_loop` 循环式闭环控制图
- 讲述重点：循环窗口让传感和反应速度不必极快，从而降低成本并提升可执行性。

## S3 系统架构：40-agent 研究平台与模型真实性审计

- 页面目的：展示 agent 分层和责任边界。
- 视觉素材：`agent_layer_map` Agent 分层图
- 讲述重点：当前共有 40 个 agent，其中前 28 个构成执行链，Agent29-40 负责项目总览、真实数据接口、成果整理、展示素材、正式 deck、实证校准门控、模型真实性审计、软传感不确定性验证、知识图谱策展、文献证据抽取、软传感保形校准和灰箱动态延迟审计。

## S4 关键证据链：从多批次瓶颈到自动重规划

- 页面目的：展示系统如何发现验证、时间和库存瓶颈，并转成资源和项目动作。
- 视觉素材：`evidence_waterfall` 瓶颈到重规划证据链
- 讲述重点：单批次成功不等于 campaign 可执行，瓶颈必须能触发重规划和写回验证。

## S5 恢复控制边界：0.75 条件恢复，0.60 失败回退

- 页面目的：统一关键工程结论，防止把条件恢复说成永久满负荷。
- 视觉素材：`recovery_boundary` 恢复进水边界图
- 讲述重点：当前恢复模式为 maintain_conditional_recovery，进水 0.75，回退 0.6。

## S6 真实数据接口：五张表连接现场和模型

- 页面目的：说明后续实证怎样落地，不停留在仿真。
- 视觉素材：`field_data_schema` 真实数据接口图
- 讲述重点：用 batch_id 把在线传感、离线检测、催化剂寿命、操作日志和成本部署数据连起来。

## S7 边界和诚实口径：研究平台，不是现场自治结论

- 页面目的：明确成熟度与尚未完成的实证验证。
- 视觉素材：`validation_boundary` 边界说明卡片
- 讲述重点：当前接口状态为 template_ready_not_field_validated，数据来源是 synthetic。

## S8 下一步：真实数据导入与参数校准

- 页面目的：给出可执行后续路线。
- 视觉素材：`calibration_roadmap` P1-P6 实证校准路线
- 讲述重点：先导入真实传感和离线标签，再校准软传感、时间预算、错峰收益和回退门槛。
